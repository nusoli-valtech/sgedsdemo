    ```
    import { exec } from 'node:child_process';

    const run = (cmd) => new Promise((resolve, reject) => exec(
      cmd,
      (error, stdout) => {
        if (error) reject(error);
        else resolve(stdout);
      },
    ));

    const changeset = await run('git diff --cached --name-only --diff-filter=ACMR');
    const modifiedFiles = changeset.split('\n').filter(Boolean);

    // check if there are any model files staged
    const modifledPartials = modifiedFiles.filter((file) => file.match(/(^|\/)_.*.json/));
    if (modifledPartials.length > 0) {
      const output = await run('npm run build:json --silent');
      console.log(output);
      await run('git add component-models.json component-definition.json component-filters.json');
    }

    // SET-03: enforce no-Publish constraint by scanning staged runtime-code files
    // for any reference to an AEM Cloud publish hostname. Skip docs/planning/config dirs.
    const PUBLISH_HOST_RE = /publish-[A-Za-z0-9-]+\.adobeaemcloud\.com/;

    const isRuntimeCodePath = (file) => {
      // Allowlist of runtime-code paths per D-05.
      if (file.startsWith('blocks/')) return true;
      if (file.startsWith('scripts/')) return true;
      // Top-level *.html / *.json only (no nested config — those are skipped).
      if (!file.includes('/')) {
        if (file.endsWith('.html')) return true;
        if (file.endsWith('.json')) return true;
      }
      return false;
    };

    const violations = [];
    for (const file of modifiedFiles) {
      if (!isRuntimeCodePath(file)) continue;
      // Read STAGED content (not working-tree content) so the guard reflects
      // exactly what is about to be committed.
      let staged;
      try {
        staged = await run(`git show :${file}`);
      } catch {
        // File staged for deletion or unreadable — nothing to scan.
        continue;
      }
      const lines = staged.split('\n');
      for (let i = 0; i < lines.length; i += 1) {
        const match = lines[i].match(PUBLISH_HOST_RE);
        if (match) {
          violations.push({ file, line: i + 1, match: match[0] });
        }
      }
    }

    if (violations.length > 0) {
      console.error('');
      console.error('pre-commit: publish-tier AEM hostnames are banned in runtime code.');
      console.error('  See ROADMAP Phase 1 SET-03 — the EDS demo intentionally has no Publish tier.');
      console.error('');
      for (const v of violations) {
        console.error(`  ${v.file}:${v.line}  matched '${v.match}'`);
      }
      console.error('');
      console.error('Fix: replace with the Author host from scripts/config.js (AEM_AUTHOR_HOST),');
      console.error('     or move the reference to .planning/ / docs/ if it is documentation.');
      process.exit(1);
    }
    ```

    Notes on intentional choices:
    - The regex `/publish-[A-Za-z0-9-]+\.adobeaemcloud\.com/` matches any AEM Cloud publish hostname (D-04 broad match). It does NOT match `author-...adobeaemcloud.com` or arbitrary URLs. Hyphens in the program/env id segment are allowed (e.g., `publish-p23458-e585661`).
    - Staged content is read via `git show :<file>` so a developer who edits the working copy after `git add` cannot bypass the check. The existing model-partials pass uses the same `--diff-filter=ACMR` namelist, so file selection is consistent.
    - Top-level HTML/JSON only (no nested directory). `paths.json`, `head.html`, `404.html`, `helix-query.yaml.json` (none currently), `package.json` are all in scope. Scanned `.json` files include `package.json` — if a future addition introduces a publish host there, it will be caught.
    - The guard explicitly skips files staged for deletion (the `try/catch` around `git show :<file>` returns silently). Deleted files cannot regress.
    - Failure message follows the D-06 prescription: file + line + matched string + one-line `why`.
    - `for...of` loop over modifiedFiles is sequential — the changeset is small (typically <20 files); no need for parallel reads. Airbnb-base permits `for...of` (only `for...in` is restricted). Suppress `no-restricted-syntax` if lint trips.
    - Quotes changed from double (`"node:child_process"`) to single (airbnb-base default). Trailing comma added after the `(error, stdout) => {...}` arrow param body to satisfy `comma-dangle: always-multiline` if the rule is in effect (existing file omits it; airbnb-base default expects it). If lint complains about the trailing comma, remove it.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Developer working tree → git index → repository | Pre-commit hook is the last automated gate; runs locally before commit lands. Bypassable via `--no-verify` (out of scope to prevent — that is a deliberate developer choice). |
| Repository → AEM runtime | Files committed without publish-host references reduce the chance of CP-1 regressing into deployed code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Tampering | Repository contents — accidental publish-host injection | mitigate | The pre-commit hook scans staged content (`git show :<file>`) — a developer cannot stage one version, edit the working copy, and slip the publish host through. Failure mode: developer stages a clean file, edits AFTER staging. The check reads STAGED content so the dirty edit is irrelevant to what gets committed; it only matters if they re-stage. |
| T-01-03-02 | Bypass | Pre-commit hook — `--no-verify` flag | accept | `git commit --no-verify` skips all hooks. This is a deliberate developer choice and out of scope for SET-03. CI-side enforcement (a future requirement, not in scope) would close this gap. Documented in CONTEXT.md as an explicit accept. |
| T-01-03-03 | Information Disclosure | Pre-commit error message | accept | The error message includes the file path, line number, and matched substring. The matched substring is a publish hostname which is itself public information (no secrets). Acceptable to surface verbatim to the developer for debuggability. |
| T-01-03-04 | Denial of Service | Pre-commit performance | mitigate | The scan reads staged content one file at a time via `git show`, scoped to runtime code paths only. Typical commit changeset is <20 files; sub-second total. Negligible DX impact. |
| T-01-03-05 | False Negative | Regex misses obfuscated publish host (e.g., string concatenation) | accept | The regex catches literal hostnames. A determined developer can defeat it by splitting the string (`'publi' + 'sh-p23458-...'`) — that is out of scope for SET-03 and would be caught at code review. Documented as a known limitation. |

This plan IS a security control. Severity is HIGH because the constraint it enforces (no-Publish architecture) is foundational to the project. Mitigation is preventive (commit gate) and detective (clear failure message naming the offender).
</threat_model>
